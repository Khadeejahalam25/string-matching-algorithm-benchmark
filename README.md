# Intelligent CV Analyzer — String Matching Algorithms

An automated CV screening tool that implements **Brute Force, Rabin-Karp, and KMP** string
matching algorithms from scratch, then uses them to extract skills from resumes and rank
candidates against job requirements — built as a Design & Analysis of Algorithms project
comparing the real-world performance of classic pattern-matching algorithms on a practical task.

## The problem

Manually screening a CV takes a recruiter roughly 6–8 minutes and is inconsistent. This project
automates skill extraction and candidate ranking using string matching, and — since that's the
actual algorithmic core of the project — rigorously benchmarks three different matching
algorithms against each other to see which is best suited for this kind of workload at scale.

## Algorithms implemented (from scratch, no library calls)

| Algorithm | Time Complexity | Approach |
|---|---|---|
| Brute Force | O(n×m) | Checks the pattern at every position in the text |
| Rabin-Karp | O(n+m) average | Rolling hash comparison, verifies on hash match |
| KMP | O(n+m) guaranteed | Preprocesses the pattern into an LPS array to skip redundant comparisons |

## How it works

1. **Parse** — extracts raw text from CVs in PDF, DOCX, or TXT format.
2. **Match** — searches the CV text for each required skill keyword using the selected
   algorithm(s), tracking match count, comparison count, and execution time per algorithm.
3. **Score** — categorizes matched/missing skills against mandatory and optional requirements
   per job role, then computes a weighted score: `Overall = (Mandatory × 0.7) + (Optional × 0.3)`.
4. **Rank & report** — sorts candidates by score, generates a ranked Excel export, and plots
   algorithm performance and score distribution charts.

Three job roles are pre-defined with mandatory/optional skill sets: **Data Scientist**,
**Web Developer**, and **Software Tester**.

## Key findings (from the full experimental run — see report)

Tested against 1000+ real resumes (PDF/DOCX/TXT):

- **KMP is ~59% faster than Brute Force** and performs **65% fewer comparisons**
- KMP's advantage grows as CV length increases, confirming its O(n+m) guarantee holds in practice
- Rabin-Karp's hashing overhead makes it the slowest in wall-clock time here, despite doing far
  fewer character-by-character comparisons than Brute Force or KMP
- **Recommendation: KMP** is best suited for production-scale CV screening — consistent
  performance, no worst-case blowup, minimal memory overhead

Full methodology, complexity analysis, and complete results are in `report/Report.pdf`.

Charts from the original full experimental run (1000+ real resumes) are included in
`report_diagrams/` for reference — aggregate performance and score statistics only.

## Evidence it runs

This repo includes a small set of **synthetic sample CVs** (`sample_cvs/`) — not real candidate
data — and the actual generated output from running the analyzer against them, in
`sample_output/`: ranked Excel results, algorithm performance comparison chart, and score
distribution chart. The original project was run against 1000+ real resumes for the report's
findings above; the synthetic set here exists purely to let anyone clone and verify the pipeline
runs end-to-end without needing access to real candidates' personal data.

## Tech stack

Python · PyPDF2 · python-docx · pandas · matplotlib · seaborn · openpyxl

## What's in this repo

- `cv_analyzer.py` — full implementation: string matching algorithms, CV parser, scoring engine,
  and report generation. (Originally built and run in Google Colab; adapted here to run as a
  standalone script.)
- `sample_cvs/` — 5 synthetic sample CVs for demo purposes.
- `sample_output/` — real output generated by running this script against the sample CVs.
- `job_descriptions/` — reference job description text files for the three supported roles.
- `report/Report.pdf` — full project report: algorithm theory, complexity analysis, experimental
  results on 1000+ resumes, and discussion.

## How to run it

```bash
git clone https://github.com/<your-username>/cv-analyzer-string-matching.git
cd cv-analyzer-string-matching

pip install -r requirements.txt
python cv_analyzer.py
```

Follow the prompts to select a job role and algorithm(s). To analyze your own CVs, replace the
contents of `sample_cvs/` with your own PDF/DOCX/TXT files. Output files
(`cv_analysis_results.xlsx` and the two chart PNGs) are generated in the directory you run the
script from — the copies in `sample_output/` are a pre-generated example from the sample CVs.
